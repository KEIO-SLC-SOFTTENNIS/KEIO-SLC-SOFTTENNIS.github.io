# 慶應 SLC ソフトテニス部 公式サイト

慶應義塾大学唯一の公認ソフトテニスサークル「SLC ソフトテニス部」の活動を紹介するための静的ウェブサイトです。GitHub Pages でホストしており、`main` ブランチにプッシュされた内容がそのまま公開されます。

---

## 公開環境
- **ホスティング:** GitHub Pages（リポジトリ直下を公開対象に設定）
- **ビルド:** なし（プレーンな HTML / CSS / JavaScript を配置するだけで公開できます）
- **推奨エディタ:** VS Code などの一般的なコードエディタ  
  ↳ Live Server 拡張機能を使うと、ブラウザで変更確認がスムーズです。

---

## ディレクトリ構成

```
.
├── index.html                # トップページ
├── pages/                    # サブページ全般
│   ├── activities/           # 活動報告（年度別 HTML）
│   ├── members/              # メンバー紹介
│   └── ...                   # その他の固定ページ
├── assets/
│   ├── css/
│   │   └── main.css          # 共通スタイル
│   ├── js/
│   │   └── main.js           # 共通スクリプト（サイドバー開閉等）
│   └── images/
│       ├── hero/             # メインビジュアル
│       ├── icons/            # アイコン類
│       ├── schedule/         # 年間予定画像
│       └── ...               # 活動・メンバー写真
└── README.md
```

---

## 更新の基本フロー

1. **画像を追加する**  
   - `assets/images/` 以下の適切なサブディレクトリに保存します。  
   - ファイル名は半角英数字とハイフン/アンダースコアのみを推奨（GitHub Pages との相性が良いため）。
2. **内容を編集する**  
   - 既存ページを修正する場合は該当 HTML/CSS/JS を編集します。  
   - Live Server などでローカル確認を行い、レイアウト崩れがないかをチェックします。
3. **新しいページを追加する**  
   - `pages/` に HTML ファイルを作成し、必要なら `pages/activities/2024.html` のように年度ごとにフォルダを切ります。  
   - ナビゲーションやサイドバーにリンクを追記し、全ページからアクセスできるようにします。
4. **リンクと画像パスを確認する**  
   - 相対パスを利用し、トップページ (`index.html`) からも参照できるか確認します。  
   - スマートフォン表示でレイアウトが崩れないか、ブラウザのデベロッパーツールで確認してください。
5. **GitHub に反映する**  
   - `git add` → `git commit` → `git push origin main`。プッシュ後、数十秒で公開が更新されます。

---

## よくある更新例

| 更新内容                 | 手順概要 |
|--------------------------|----------|
| 活動報告を 1 ページ追加 | `pages/activities/` に HTML を複製 → 新年度の内容へ書き換え → 一覧ページとサイドバーにリンクを追加 |
| メンバー紹介を更新       | `pages/members/` 配下の HTML を編集し、新しい画像とテキストを差し替え |
| 年間スケジュール画像更新 | `assets/images/schedule/` に新画像を配置し、該当ページの `<img>` パスを変更 |
| トップの告知を変更       | `index.html` の告知セクションを編集。必要に応じて CSS でスタイル調整 |

## コーディングガイド
- **HTML:** 既存のセクション構造（`section` / `article` など）を参考に、見出し (`h2`/`h3`) を整理して追加してください。
- **CSS:** 共通スタイルは `assets/css/main.css` にまとまっています。追加スタイルが多い場合はコメントで区切ると保守しやすくなります。
- **JavaScript:** 基本的にサイドバーの開閉や簡単なインタラクションのみを `assets/js/main.js` に記述しています。ライブラリは使用していません。
- **画像:** 5MB 以下を推奨。解像度は 2000px 程度までにすると読み込みが軽くなります。

## Tips
- **ブランチ運用**: 大きな変更時は `feature/◯◯` ブランチで作業し、動作確認後に `main` へマージすると安全です。
- **リンク切れチェック**: ブラウザで `F12` → Console を開き、404 が出ていないか確認してください。
- **バックアップ**: 画像など大きな変更を行う前に別ブランチやクラウドにコピーしておくと安心です。

不明点や改善案があれば、Issue や Pull Request で共有してください。
